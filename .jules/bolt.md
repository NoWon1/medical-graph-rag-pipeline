## 2025-02-14 - Vectorized MMR Reranking via NumPy
**Learning:** The previous implementation of `mmr_rerank` in `cancer_retrieval.py` used a pure Python `_cosine` function based on loops and `math.sqrt`. This is extremely slow for high-dimensional vectors (like `HuggingFaceEmbeddings` with 768 dimensions), especially when calculating pairwise similarities during the MMR selection process. The pure Python loops caused massive overhead when dealing with candidate sizes from reciprocal rank fusion (up to 60 docs).
**Action:** Next time, identify loops doing heavy mathematical vector operations (like dot products and cosine similarities) on large document chunks and replace them with NumPy's vectorized operations (`np.dot`, `np.linalg.norm`). This can yield a 20x or more speedup. Ensure the vectors are cast to NumPy arrays at the beginning of the function to maximize efficiency.
